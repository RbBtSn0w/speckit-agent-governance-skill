# Contributing to Speckit Agent Governance Skill

Thank you for your interest in contributing to this project!

## Development Workflow

1.  **Fork the repository** and create your branch from `main`.
2.  **Follow Conventional Commits**: We use semantic-release, so please ensure your commit messages follow the [Conventional Commits](https://www.conventionalcommits.org/) specification.
    -   `feat: ...` for new features
    -   `fix: ...` for bug fixes
    -   `docs: ...` for documentation changes
    -   `chore: ...` for maintenance tasks
3.  **Test your changes**: Ensure that your changes do not break existing functionality.
4.  **Submit a Pull Request**: Provide a clear description of the changes and link any related issues.

## Code of Conduct

Please be respectful and professional in all interactions within this project.
